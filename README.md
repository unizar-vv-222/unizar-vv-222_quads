# G222 Quads

Proyecto desarrollado para la asignatura de Verificación y Validación.

La aplicación consiste en un sistema Android para la gestión de quads y reservas, incluyendo funcionalidades de administración de quads, creación y gestión de reservas y envío de información de reservas.

---

# Contenido del repositorio

## `g222_quads/`

Contiene el proyecto completo de Android Studio.

Incluye:
* Código fuente de la aplicación
* Activities y Fragments para la interfaz gráfica
* Base de datos y repositorios
* ViewModels para la gestión del estado
* Recursos XML para layouts y estilos
* Pruebas unitarias e instrumentadas
* Configuración Gradle del proyecto

Para ejecutar el proyecto:

1. Abrir la carpeta `g222_quads` desde Android Studio.
2. Sincronizar dependencias Gradle.
3. Ejecutar la aplicación en un emulador o dispositivo Android.

---

## `quads_modelo.vpp`

Archivo del modelo de diseño de la aplicación realizado con Visual Paradigm.

Incluye distintos diagramas utilizados durante el análisis y diseño del sistema, como:

* Diagramas de clases
* Diagramas de casos de uso
* Diagramas de componentes
* Diagramas de despliegue
* Otros modelos relacionados con la arquitectura y diseño de la aplicación

# Gestión de incidencias

El repositorio incluye una plantilla de incidencias (*Issue Template*) para facilitar el registro y seguimiento de errores, mejoras y nuevas funcionalidades.

Su uso permite:

* Mantener una estructura homogénea en las incidencias
* Facilitar la trazabilidad de defectos
* Documentar correctamente errores detectados durante las pruebas
* Mejorar la organización y seguimiento del proyecto

Para crear una nueva incidencia:

1. Acceder a la pestaña **Issues** del repositorio.
2. Pulsar en **New Issue**.
3. Seleccionar la plantilla disponible.
4. Completar la información solicitada.
